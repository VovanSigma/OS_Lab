студента ІПС-32

Варіант роботи

Комунікація через статус завершення процесу (C++). 
Підхід до обробки: опитування (polling).
Компонентами обчислень є окремі процеси fn1 та fn2, які запускаються головним процесом Mn. 
Операції: сума дійсних значень, побітова диз'юнкція статусів, конкатенація результатів.

Опис реалізації

У завданні потрібно реалізувати систему Mn, яка запускає два дочірні процеси fn1 і fn2. 
Кожен процес виконує обчислення для переданого аргументу x та повертає свій результат через код завершення (1 байт). 
У цьому байті зберігаються біти статусу (OK, FAIL, UNDEFINED) та маленьке числове значення.

Головний процес створює дочірні процеси за допомогою CreateProcessA, 
постійно опитує їхній стан за допомогою WaitForSingleObject і після завершення отримує exit code. 
Потім код декодується у статус та значення. 
Після цього виконується побітова диз'юнкція статусів та обчислення суми отриманих значень.

fn1 та fn2 мають власні правила обчислення і можуть повертати три можливі стани:
ST_OK, ST_FAIL, ST_UNDEF.

Як працювати

Головний процес запускається як стандартна консольна програма.
Після запуску користувач вводить ціле значення x. 
Для кожного введеного значення створюються два дочірні процеси fn1 та fn2, які виконують відповідні обчислення і повертають закодований статус завершення. 
Після завершення обох процесів програма декодує отримані статуси, визначає наявність результатів, обчислює побітову диз'юнкцію статусів та суму отриманих значень, після чого виводить їх у консоль.
Введення символу q здійснює коректне завершення роботи програми.
